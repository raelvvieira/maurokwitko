

## Plan: Persistir dados com Lovable Cloud (Supabase)

Atualmente todos os dados (álbuns, cursos, blog, materiais, e-books, livros, descontos) ficam apenas no state React e são perdidos ao recarregar a página. Vamos criar tabelas no banco de dados e conectar a aplicação para salvar e carregar tudo automaticamente.

### 1. Habilitar Lovable Cloud e criar tabelas

Criar as seguintes tabelas via migrations:

- **albums** (`id uuid PK, title text, cover_color text, created_at timestamptz`)
- **album_tracks** (`id uuid PK, album_id uuid FK→albums, title text, youtube_url text, position int`)
- **course_categories** (`id uuid PK, name text, created_at timestamptz`)
- **course_videos** (`id uuid PK, category_id uuid FK→course_categories, title text, youtube_id text, description text, position int`)
- **blog_posts** (`id uuid PK, title text, content text, image_url text, author text, created_at timestamptz`)
- **materials** (`id uuid PK, title text, type text, size text, url text, created_at timestamptz`)
- **ebooks** (`id uuid PK, title text, author text, description text, pages int, url text`)
- **livros** (`id uuid PK, title text, author text, description text, pages int, url text`)
- **discounts** (`id uuid PK, title text, description text, percentage int, course_ids text[], coupon_code text, expires_at date, active boolean`)

All tables will have RLS disabled initially (no auth yet) so all operations work without login.

### 2. Create data hooks

New file `src/hooks/useSupabaseData.ts` with custom hooks per entity using `@tanstack/react-query` + Supabase client:
- `useAlbums()`, `useCourseCategories()`, `useBlogPosts()`, `useMaterials()`, `useEbooks()`, `useLivros()`, `useDiscounts()`
- Each hook returns `{ data, isLoading, add, remove }` functions that do real DB operations and invalidate queries

### 3. Update AppContext

Replace the `useState` arrays with data from the Supabase hooks. The context will proxy the hook data so all existing pages continue working without changes. The `add*` and `remove*` functions will call Supabase instead of local state.

### 4. Seed initial data

Insert the current default mock data into the database so the platform starts with content.

### Files to create/modify

| File | Action |
|------|--------|
| Migration SQL | Create all 9 tables |
| `src/integrations/supabase/client.ts` | Auto-generated by Lovable Cloud |
| `src/hooks/useSupabaseData.ts` | New — all CRUD hooks |
| `src/context/AppContext.tsx` | Replace useState with Supabase hooks |
| Seed SQL | Insert default albums, categories, books, etc. |

### Technical note
- No auth for now — tables are public read/write. Auth + RLS can be added later.
- Album tracks and course videos use separate tables with foreign keys for proper relational data.
- Images/files still use URLs (no file upload to storage yet).
